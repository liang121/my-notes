# Notes 目录工作规则

这是老板的笔记归档目录。老板会发链接（YouTube、文章、文档等）进来，要求总结/转录后落档到本目录的子目录中。

## 归档规则

1. **按主题分子目录归档**：每个主题对应一个子目录（如 `vibecoding/`、`react/`、`ai-agents/` 等）。落档文件放进对应子目录，不要堆在根目录。

2. **判断归属子目录**：
   - 老板明确指定子目录 → 直接放进去
   - 老板**没有**明确指定 → **必须先问**："这份材料归到哪个子目录？"，列出现有子目录供选择，不要自作主张
   - 推断的归属也要先确认，不要默认就放

3. **子目录不存在时**：
   - 不要自动创建
   - 先问老板："`xxx/` 子目录不存在，要新建吗？"
   - 得到确认后再 mkdir 并落档

4. **文件命名**：用描述性的中文或英文短名 + `.md`，不要带日期前缀（除非老板要求）。

## 落档内容格式

除非老板另有要求，落档的 markdown 文件应包含：
- 原始链接（放在文件开头）
- 来源/作者（如能识别）
- 核心总结
- 关键要点 / 时间戳（视频类）
- 老板可能关心的延伸思考

## 工作流提醒

- 总结视频优先用 `youtube-summary` skill（轻量、快），拉不到字幕再 fallback 到 `video-summarizer`
- 落档前先确认子目录归属，**永远不要在没确认归属的情况下直接写文件**
